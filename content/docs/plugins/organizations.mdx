## Overview

The Organizations plugin provides multi-tenancy and team management capabilities. It enables users to create organizations, manage members with roles, organize members into teams, and send email-based invitations.

### Core Entities

- **Organizations**: Owner-based organizational units
- **Members**: Users assigned to organizations with roles
- **Teams**: Groups of members within organizations
- **Invitations**: Email-based, expiring invitation tokens

### Authorization

Only organization owners can manage organizations, members, teams, and invitations.

### Features

- Multi-tenant organization support with owner-based authorization
- Role-based member management within organizations
- Team organization and management within organizations
- Email-based member invitations with configurable expiration
- Team member management
- Full lifecycle management (create, read, update, delete operations)

## Configuration

`Standalone Mode:`

```toml
[organizations]
enabled = true
invitation_expires_in = "168h"
```

`Library Mode:`

```go
pluginConfig := &types.OrganizationsPluginConfig{
  Enabled:             true,
  InvitationExpiresIn: 7 * 24 * time.Hour,
  DatabaseHooks: &types.OrganizationsDatabaseHooksConfig{
    Organizations: &types.OrganizationDatabaseHooksConfig{
      BeforeCreate: customBeforeCreateOrganization,
      AfterCreate:  customAfterCreateOrganization,
    },
    Members: &types.OrganizationMemberDatabaseHooksConfig{
      AfterCreate: customAfterAddMember,
    },
    Invitations: &types.OrganizationInvitationDatabaseHooksConfig{
      AfterCreate: customAfterInvitationCreated,
    },
    Teams: &types.OrganizationTeamDatabaseHooksConfig{
      BeforeCreate: customBeforeCreateTeam,
    },
    TeamMembers: &types.OrganizationTeamMemberDatabaseHooksConfig{
      AfterCreate: customAfterAddTeamMember,
    },
  },
}

plugin := organizations.New(pluginConfig)
```

## Plugin Capabilities

The Organizations plugin provides hooks and capabilities for customizing organization lifecycle events and member management.

### Database Hooks

Organizations plugin supports the following database hooks:

- `organizations.before_create`: Execute custom logic before organization creation
- `organizations.after_create`: Execute custom logic after organization creation
- `organization_members.after_create`: Execute custom logic after member is added to organization
- `organization_invitations.after_create`: Execute custom logic after invitation is created
- `organization_teams.before_create`: Execute custom logic before team creation
- `organization_team_members.after_create`: Execute custom logic after member is added to team

#### Example Usage

`Standalone mode:`

```toml
[[hooks]]
event = "organizations.after_create"
handler = "send_welcome_email"

[[hooks]]
event = "organization_members.after_create"
handler = "notify_member_added"

[[hooks]]
event = "organization_invitations.after_create"
handler = "send_invitation_email"
```

`Library mode:`

```go
import (
  organizationsplugin "github.com/Authula/authula/plugins/organizations"
)

// Define hook handlers
func customAfterCreateOrganization(ctx context.Context, org *models.Organization) error {
  // Send welcome email to organization owner
  return sendWelcomeEmail(org.Owner.Email)
}

func customAfterAddMember(ctx context.Context, member *models.OrganizationMember) error {
  // Notify organization owner that a new member was added
  return notifyMemberAdded(member.OrganizationID, member.UserID)
}

func customAfterInvitationCreated(ctx context.Context, invitation *models.OrganizationInvitation) error {
  // Send invitation email
  return sendInvitationEmail(invitation.Email, invitation.InvitationToken)
}

// Register hooks with plugin config
config := &types.OrganizationsPluginConfig{
  Enabled: true,
  DatabaseHooks: &types.OrganizationsDatabaseHooksConfig{
    Organizations: &types.OrganizationDatabaseHooksConfig{
      AfterCreate: customAfterCreateOrganization,
    },
    Members: &types.OrganizationMemberDatabaseHooksConfig{
      AfterCreate: customAfterAddMember,
    },
    Invitations: &types.OrganizationInvitationDatabaseHooksConfig{
      AfterCreate: customAfterInvitationCreated,
    },
  },
}

plugin := organizationsplugin.New(config)
```
