<Steps>
<Step>

### Install the Package

```bash
$ go get "github.com/GoBetterAuth/go-better-auth"
```

</Step>

<Step>

### Set environment variables

Create a `.env` file in the root of your project and add the following environment variables:

```txt title=".env"
# The base URL of your API
GO_BETTER_AUTH_BASE_URL=

# Used for encryption
GO_BETTER_AUTH_SECRET=
```

</Step>

<Step>

### Create auth config

```go
import (
	gobetterauth "github.com/GoBetterAuth/go-better-auth"
	gobetterauthdomain "github.com/GoBetterAuth/go-better-auth/pkg/domain"
)

config := gobetterauthdomain.NewConfig(
  // Add your config options here
)
// You can pass in an existing *gorm.DB instance as the second argument to use that instead of having the library create a new one
auth, err := gobetterauth.New(config, nil)
```

</Step>

<Step>

### Configure database

Go Better Auth supports multiple database providers such as SQLite, Postgres and more coming soon. You can select a database provider by supplying it in the config.

<CodeBlockTabs defaultValue="sqlite" groupId="database-provider" persist>
    <CodeBlockTabsList>

      <CodeBlockTabsTrigger value="postgres">

    <CodeBlockTab value="sqlite">
      ```go
      gobetterauthdomain.NewConfig(
        gobetterauthdomain.WithDatabase(gobetterauthdomain.DatabaseConfig{
          Provider:         "sqlite",
          ConnectionString: "./auth.db", // Path to your sqlite database file
        }),
      )
      ```
    </CodeBlockTab>

    <CodeBlockTab value="postgres">
      ```go
      gobetterauthdomain.NewConfig(
        gobetterauthdomain.WithDatabase(gobetterauthdomain.DatabaseConfig{
          Provider:         "postgres",
          ConnectionString: os.Getenv("DATABASE_URL"), // Add this ENV variable to your .env
        }),
      )
      ```
    </CodeBlockTab>

  </CodeBlockTabs>

</Step>

<Step>

### Run database migrations

Go Better Auth provides migration files which you can run directly against your database of choice. Or you can run the migrations programmatically by running the `RunMigrations()` function. An example is shown below:

```go
// Create the config
config := gobetterauthdomain.NewConfig()
auth, err := gobetterauth.New(config, nil)

// Run the migrations using the config
auth.RunMigrations(ctx, config)
```

</Step>

<Step>

### Authentication Methods

Configure the authentication methods you want to use in your config struct. Go Better Auth supports the basic Email/Password authentication method and will soon support social sign-on providers such as Google, GitHub, Discord and more.

```go
config := gobetterauthdomain.NewConfig(
  gobetterauthdomain.WithEmailPassword(gobetterauthdomain.EmailPasswordConfig{
    Enabled:                  true,
    RequireEmailVerification: true,
    AutoSignIn:               true,
  }),
)
```

</Step>

<Step>

### Mount Handlers

To handle API requests, you need to integrate the Go Better Auth handlers in your API.

This is an example of how it can be done with the standard library server:

```go
// Initialise auth
config := gobetterauthdomain.NewConfig()
auth, err := gobetterauth.New(config, nil)
if err != nil {
  log.Fatal(err)
}

// Mount auth routes
http.Handle("/auth/", auth.Handler())

// Start the server
log.Println("Server running on http://localhost:8080")
log.Fatal(http.ListenAndServe(":8080", nil))
```

Take a look at the [Examples](/docs/examples/routers) section to see how to integrate it with other frameworks.

</Step>
