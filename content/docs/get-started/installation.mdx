There are two ways to configure GoBetterAuth in your project: as a standalone server or as a library integrated into your existing API server.

The following sections will guide you through both methods.

## Standalone Mode

<Steps>
<Step>

### Setup configuration file

- Make a copy of the `config.example.toml` file provided in the repository.
- Rename the copied file to `config.toml`.
- Populate the configuration fields with your desired settings. Most of these settings allow you to configure the server according to your needs.
- It is recommended that you do not set secret values within the `config.toml` file directly. Instead, use environment variables to manage sensitive information securely.

```toml title="config.toml"
app_name = "GoBetterAuth"
base_url = "http://localhost:8080"
base_path = "/auth"

[database]
provider = "postgres"

# ... other settings
```

</Step>

<Step>

### Set environment variables via .env file

- Make a copy of the `.env.example` file provided in the repository.
- Rename the copied file to `.env`.
- Populate the environment variables with your desired values. These values will override any corresponding values set in the `config.toml` file.

```txt title=".env"
# Path to your configuration file (e.g., ./config.toml) (optional)
GO_BETTER_AUTH_CONFIG_PATH=

# The base URL of your API
GO_BETTER_AUTH_BASE_URL=

# Used for encryption. Generate using `openssl rand -base64 50 | tr -dc 'A-Za-z0-9' | cut -c -40`
GO_BETTER_AUTH_SECRET=

# sqlite/postgres/mysql connection string
GO_BETTER_AUTH_DATABASE_URL=
```

</Step>

<Step>

### Run the server via Docker

```bash
$ docker run -itd \
  -p 8080:8080 \
  -v $(pwd)/config.toml:/home/appuser/config.toml \
  -e GO_BETTER_AUTH_BASE_URL=http://localhost:8080 \
  -e GO_BETTER_AUTH_SECRET=my-app-secret \
  -e GO_BETTER_AUTH_DATABASE_URL=<your_connection_string> \
  ghcr.io/gobetterauth/go-better-auth:latest
```

</Step>

</Steps>

## Library Mode

<Steps>
<Step>

### Set environment variables

Create a `.env` file in the root of your project and add the following environment variables:

```txt title=".env"
# The base URL of your API
GO_BETTER_AUTH_BASE_URL=

# Used for encryption. Generate using `openssl rand -base64 50 | tr -dc 'A-Za-z0-9' | cut -c -40`
GO_BETTER_AUTH_SECRET=

# sqlite/postgres/mysql connection string
GO_BETTER_AUTH_DATABASE_URL=
```

</Step>

<Step>

### Install the Package

```bash
$ go get github.com/GoBetterAuth/go-better-auth/v2
```

</Step>

<Step>

### Create auth config

```go
import (
  gobetterauth "github.com/GoBetterAuth/go-better-auth/v2"
  gobetterauthconfig "github.com/GoBetterAuth/go-better-auth/v2/config"
)

config := gobetterauthconfig.NewConfig(/*...*/)
auth := gobetterauth.New(&gobetterauth.AuthConfig{
  Config:  config,
  Plugins: [/*...*/],
})
```

</Step>

<Step>

<Step>

### Database migrations

GoBetterAuth runs all migrations (core & plugins) automatically when you create a new instance of the library so all the relevant database tables are created for you without any additional steps. This is done via [dbmate](https://github.com/amacneil/dbmate) and migration files are embedded in the binary. But GoBetterAuth also provides migration files which you can run manually against your database which you can find within the `migrations/` directories within the repository.

</Step>

<Step>

### Authentication Methods

Configure the authentication methods you want to use by choosing from the available plugins. GoBetterAuth supports the Email/Password authentication method as a plugin and also OAuth2 providers such as Discord, GitHub and Google with more coming soon.

```go
import (
  gobetterauth "github.com/GoBetterAuth/go-better-auth/v2"
  gobetterauthconfig "github.com/GoBetterAuth/go-better-auth/v2/config"
  emailpasswordplugin "github.com/GoBetterAuth/go-better-auth/v2/plugins/email-password"
  emailpasswordplugintypes "github.com/GoBetterAuth/go-better-auth/v2/plugins/email-password/types"
)

config := gobetterauthconfig.NewConfig(/*...*/)
auth := gobetterauth.New(&gobetterauth.AuthConfig{
  Config:  config,
  Plugins: [
    emailpasswordplugin.New(emailpasswordplugintypes.EmailPasswordPluginConfig{
      Enabled:                  true,
      MinPasswordLength:        8,
      MaxPasswordLength:        32,
      DisableSignUp:            false,
      RequireEmailVerification: true,
      AutoSignIn:               true,
      SendEmailOnSignUp:        true,
    }),
  ],
})
```

</Step>

<Step>

### Mount Handlers

To handle API requests, you need to integrate the GoBetterAuth handlers in your API.

This is an example of how it can be done with the standard library server:

```go
// Initialise auth
config := gobetterauthconfig.NewConfig(/*...*/)
auth := gobetterauth.New(&gobetterauth.AuthConfig{
  Config:  config,
  Plugins: [/*...*/],
})

// Start the server and mount the handler
log.Fatal(http.ListenAndServe(":8080", auth.Handler()))
```

Take a look at the [Integrations](/docs/integrations/routers) section to see how to integrate it with other frameworks.

</Step>

</Steps>
